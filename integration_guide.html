1. What We Built

Modernized the print-only checkout experience into a digital-first, paperless workflow.

Before

Standard Print Billing

  • Shopkeeper enters item metrics.
  • Presses the SHIFT key trigger.
  • Standard browser window print dialog opens.
  • Transaction parameters saved into MongoDB.
  • Zero digital invoice copy sent to the client.
After (WhatsApp Active)

Modern Digital Checkout

  • Input phone number and customer name first.
  • Pressing SHIFT prints receipt physical slip.
  • Transaction details saved securely in database.
  • Server auto-generates a formatted PDF invoice dynamically.
  • PDF uploaded & sent to the customer's WhatsApp instantly.

NA Bypass: If the shopkeeper types "NA" or leaves the phone number blank, the WhatsApp pipeline is safely bypassed. The app functions in traditional print-only mode.

2.2 — Authentication

Every transaction query to Meta's servers requires authorization headers using a secure Bearer token:

HTTP Header 
Authorization: Bearer EAATDQj22ZCs...

AccessToken Types:

  • Temporary Token: Valid for 24 hours. Excellent for active localized testing.
  • System User Token: Permanent credential generated via Business Manager. Mandatory for stable production environments.

2.3 — Media Upload & Message Template Dispatch (Two-Step Process)

To deliver a file attachment via WhatsApp Cloud API, you must complete two steps: first upload the binary file to Meta's endpoints, then reference the returned ID in the template message call.

Upload the raw PDF buffer to Meta media repositories:

POST Request 
POST https://graph.facebook.com/v25.0/{PHONE_NUMBER_ID}/media
Content-Type: multipart/form-data
Authorization: Bearer YOUR_TOKEN

Fields:
 - messaging_product: "whatsapp"
 - file: 
 - type: "application/pdf"

Response: { "id": "1234567890" }  <-- media_id

Dispatch the message referencing the newly created media_id:

JSON Payload 
POST https://graph.facebook.com/v25.0/{PHONE_NUMBER_ID}/messages
Content-Type: application/json
Authorization: Bearer YOUR_TOKEN

{
  "messaging_product": "whatsapp",
  "to": "919899258797",
  "type": "template",
  "template": {
    "name": "perfectcollection_invoice",
    "language": { "code": "en_US" },
    "components": [
      {
        "type": "header",
        "parameters": [
          { "type": "document", "document": { "id": "1234567890", "filename": "Invoice.pdf" } }
        ]
      },
      {
        "type": "body",
        "parameters": [
          { "type": "text", "text": "John" },
          { "type": "text", "text": "#42" }
        ]
      }
    ]
  }
}

2.4 — Understanding Templates

Meta mandates pre-approved templates for outgoing business messages to reduce spam. Our template perfectcollection_invoice uses the following schema:

Header: Document attachment (the dynamic PDF invoice).

Body: Hello {{1}}, Your invoice for order {{2}} is attached. Thank you for shopping with us!

Parameters map dynamically: {{1}} is Customer Name, and {{2}} is Invoice Number.

2.5 — Phone Numbers Formats

The API expects numbers in full international layout without the + prefix or empty spaces (e.g. 919899258797). The application script detects 10-digit entries and automatically prefixes India's code 91.

    const PDFDocument = require("pdfkit");

/**
 * Generates a PDF buffer from transaction data.
 * Returns a Promise that resolves to a Buffer containing the PDF bytes.
 */
function generateInvoicePdf(transaction) {
  return new Promise((resolve, reject) => {
    const doc = new PDFDocument({ margin: 40, size: "A4" });
    const chunks = [];

    doc.on("data", (chunk) => chunks.push(chunk));
    doc.on("end", () => resolve(Buffer.concat(chunks)));
    doc.on("error", reject);

    // --- Header ---
    doc.fontSize(20).font("Helvetica-Bold").text("Perfect Collection", { align: "center" });
    doc.fontSize(10).font("Helvetica").text("(9899258797)", { align: "center" });
    doc.fontSize(9).text("SC-190, Jaipuria Sunrise Plaza, Indirapuram, GIZB, UP", { align: "center" });
    doc.moveDown();

    // --- Invoice info ---
    doc.fontSize(11).font("Helvetica-Bold").text(`Estimate No: ${transaction.transactionNumber}`);
    doc.font("Helvetica").text(`Date: ${transaction.createdAt}`);
    doc.text(`Payment: ${transaction.transactionType}`);
    if (transaction.customerName) {
      doc.text(`Customer: ${transaction.customerName}`);
    }
    doc.moveDown();

    // --- Items table ---
    doc.font("Helvetica-Bold");
    const tableTop = doc.y;
    doc.text("Item", 40, tableTop, { width: 150 });
    doc.text("Price", 190, tableTop, { width: 80, align: "right" });
    doc.text("Qty", 270, tableTop, { width: 60, align: "right" });
    doc.text("Total", 330, tableTop, { width: 80, align: "right" });
    doc.moveTo(40, doc.y + 2).lineTo(410, doc.y + 2).stroke();
    doc.moveDown(0.5);

    doc.font("Helvetica");
    for (const item of transaction.purchases) {
      const y = doc.y;
      doc.text(item.itemName, 40, y, { width: 150 });
      doc.text(String(item.itemPrice), 190, y, { width: 80, align: "right" });
      doc.text(String(item.itemQuantity), 270, y, { width: 60, align: "right" });
      doc.text(String(item.itemTotalPrice), 330, y, { width: 80, align: "right" });
      doc.moveDown(0.3);
    }

    doc.moveTo(40, doc.y + 2).lineTo(410, doc.y + 2).stroke();
    doc.moveDown();

    // --- Totals ---
    doc.font("Helvetica-Bold").fontSize(12);
    doc.text(`Total Items: ${transaction.totalItems}`, { align: "right" });
    doc.text(`Net Total: Rs.${transaction.netPrice}`, { align: "right" });
    doc.moveDown(2);

    // --- Footer ---
    doc.fontSize(10).font("Helvetica").text("FIXED PRICE SHOP", { align: "center" });
    doc.text("Thank you for your purchase!", { align: "center" });

    doc.end();
  });
}

    const axios = require("axios");
const FormData = require("form-data");

const GRAPH_API_URL = "https://graph.facebook.com";

/**
 * Formats a phone number for WhatsApp API.
 * - If 10 digits, prepends India country code 91
 * - Strips any leading + or spaces
 */
function formatPhoneNumber(phone) {
  var cleaned = phone.replace(/[\s\-\+]/g, "");
  if (cleaned.length === 10) {
    cleaned = "91" + cleaned;
  }
  return cleaned;
}

/**
 * Uploads a PDF buffer to Meta's media API and returns the media ID.
 */
async function uploadPdf(pdfBuffer, filename) {
  var version = process.env.GRAPH_API_VERSION || "v25.0";
  var phoneNumberId = process.env.WHATSAPP_PHONE_NUMBER_ID;
  var token = process.env.WHATSAPP_TOKEN;

  var form = new FormData();
  form.append("messaging_product", "whatsapp");
  form.append("file", pdfBuffer, {
    filename: filename,
    contentType: "application/pdf",
  });
  form.append("type", "application/pdf");

  var response = await axios.post(
    `${GRAPH_API_URL}/${version}/${phoneNumberId}/media`,
    form,
    {
      headers: {
        Authorization: `Bearer ${token}`,
        ...form.getHeaders(),
      },
    }
  );

  return response.data.id;
}

/**
 * Sends a PDF invoice to a WhatsApp number using the
 * "perfectcollection_invoice" approved template.
 */
async function sendInvoice(phoneNumber, pdfBuffer, filename, customerName, invoiceNumber) {
  var formattedPhone = formatPhoneNumber(phoneNumber);
  var mediaId = await uploadPdf(pdfBuffer, filename);

  var version = process.env.GRAPH_API_VERSION || "v25.0";
  var phoneNumberId = process.env.WHATSAPP_PHONE_NUMBER_ID;
  var token = process.env.WHATSAPP_TOKEN;

  var response = await axios.post(
    `${GRAPH_API_URL}/${version}/${phoneNumberId}/messages`,
    {
      messaging_product: "whatsapp",
      to: formattedPhone,
      type: "template",
      template: {
        name: "perfectcollection_invoice",
        language: { code: "en_US" },
        components: [
          {
            type: "header",
            parameters: [
              {
                type: "document",
                document: {
                  id: mediaId,
                  filename: filename,
                },
              },
            ],
          },
          {
            type: "body",
            parameters: [
              {
                type: "text",
                text: customerName || "Customer",
              },
              {
                type: "text",
                text: String(invoiceNumber || "N/A"),
              },
            ],
          },
        ],
      },
    },
    {
      headers: {
        Authorization: `Bearer ${token}`,
        "Content-Type": "application/json",
      },
    }
  );

  return response.data;
}

module.exports = { sendInvoice, formatPhoneNumber };

    const Transaction = require("../models/transaction");
const { generateInvoicePdf } = require("../services/pdfGenerator");

var moment = require("moment");

module.exports.create = async function (req, res) {
  var x = req.body;
  var myPurchases = [];
  for (var i = 0; i < x.purchases.length; i++) {
    var ob = {};
    ob["itemNumber"] = x.purchases[i].itemNumber;
    ob["itemName"] = x.purchases[i].itemName;
    ob["itemPrice"] = x.purchases[i].itemPrice;
    ob["itemQuantity"] = x.purchases[i].itemQuantity;
    ob["itemTotalPrice"] = x.purchases[i].itemTotalPrice;
    myPurchases.push(ob);
  }

  try {
    var transaction = await Transaction.create({
      transactionNumber: parseInt(x.transactionNumber),
      transactionName: x.customerName || "TBD",
      customerName: x.customerName || "",
      phoneNumber: x.phoneNumber || "",
      transactionType: x.transactionType,
      remarks: x.remarks,
      shopname: x.shopname,
      purchases: myPurchases,
      totalItems: parseInt(x.totalItems),
      totalPrice: parseInt(x.totalPrice),
      netPrice: parseInt(x.netPrice),
      gstAsPerfive: x.gstAsPerfive ? parseFloat(x.gstAsPerfive) : 0,
      gstAsPertwel: x.gstAsPertwel ? parseFloat(x.gstAsPertwel) : 0,
      createdAt: moment(x.createdAt),
    });

    console.log("Transaction saved:", transaction.transactionNumber);

    // --- WhatsApp Invoice Pipeline ---
    var phoneNumber = x.phoneNumber;
    var skipWhatsApp = !phoneNumber || phoneNumber.toUpperCase() === "NA" || phoneNumber.trim() === "";
    var whatsappStatus = "skipped";

    if (!skipWhatsApp) {
      try {
        var pdfBuffer = await generateInvoicePdf({
          transactionNumber: transaction.transactionNumber,
          createdAt: x.createdAt,
          transactionType: x.transactionType,
          customerName: x.customerName,
          purchases: myPurchases,
          totalItems: transaction.totalItems,
          netPrice: transaction.netPrice,
        });

        var filename = "PerfectCollection_Invoice_" + transaction.transactionNumber + ".pdf";

        await sendInvoice(phoneNumber, pdfBuffer, filename, x.customerName || "Customer", transaction.transactionNumber);
        whatsappStatus = "sent";
        console.log("WhatsApp invoice sent to " + phoneNumber);
      } catch (waError) {
        whatsappStatus = "failed";
        console.error("WhatsApp dispatch error:", waError.response ? waError.response.data : waError.message);
        // Best effort: Failure here does not block MongoDB save or printing.
      }
    }

    return res.json({
      success: true,
      transactionId: transaction._id,
      whatsappStatus: whatsappStatus,
    });
  } catch (err) {
    console.log("Error in creating a transaction:", err);
    return res.status(500).json({ success: false, error: err.message });
  }
};

    6. The Complete Flow — Client & Server Lifecycle

    Below is the chronological path taken by code modules when a billing operation occurs:

    Step 1 Client browser

    Initialize Interface

    Shopkeeper navigates to checkout endpoint. Front-end focuses cursor on #phone-input.

    Step 2 Client browser

    Input Information

    Shopkeeper enters phone → hits Enter → enters name → hits Enter → enters item details recursively.

    Step 3 Client browser

    Trigger Billing

    Shopkeeper presses the SHIFT key, which runs validations, strips inputs, starts browser printing, and sends a JSON payload to the server.

    Step 4 Node.js Server

    Server Processing

    Server executes transaction logic in the backend:

    Save DB: Saves billing record data in MongoDB with phone and customer details.
    Skip Check: Evaluates if phone is "NA" or empty. If so, skips WhatsApp steps.
    Build PDF: Calls pdfGenerator to create the A4 invoice buffer in-memory.
    Upload to Meta: Sends the PDF buffer to Meta's media upload API to get a media_id.
    Send Message: Calls Meta's message API to send the template message with the invoice PDF.
    Step 5 Client browser

    Completion

    Client browser processes server response, logs the WhatsApp delivery state, and reloads the checkout page.

    7. Errors We Hit and How We Fixed Them

    Reference issues encountered during build testing and their fixes:

    Error 1: Parameter Mismatch

    details: "body: number of localizable_params (1) does not match (2)"
    Cause: Message template expected both name and order number variables, but we only sent name.
    Fix: Added the invoice ID parameter to the API request payload.

    Error 2: Missing Button Param

    details: "buttons: Button at index 0 of type Url requires a parameter"
    Cause: Template had a dynamic URL button but the request payload didn't include the button parameter.
    Fix: Simplified the template in WhatsApp Manager to remove the dynamic URL button.

    Error 3: Template Pending Review

    details: "template name does not exist in en_US"
    Cause: Calling API with a template name that is still under review or disabled in Meta dashboard.
    Fix: Wait for Meta template approval (typically takes 1-5 minutes).

    Error 4: Expired Token

    code: 190 (Authentication Error)
    Cause: Temporary developers console access tokens expire automatically in 24 hours.
    Fix: Generate a new token, or configure a permanent System User Token.

    Error 5: Total calculation is NaN

    totalItems: NaN, totalPrice: NaN
    Cause: Billing was triggered with an empty shopping cart.
    Fix: Added front-end validation check to ensure at least one item exists in checkout.

    Error 6: DB Authorization Failure

    AtlasError: "not authorized on admin to execute command"
    Cause: MongoDB connection string was missing the target database name, so it defaulted to admin database.
    Fix: Explicitly add database routing path /perfectCollectionDb in MONGO_URI.

    8. Next Steps to Go to Production

    Follow these steps to move from local development testing to active production use:

    8.1 — Get a Permanent System Access Token

    Temporary developers tokens expire after 24 hours. To generate a permanent token:

    1. Go to Business Settings → System Users.
    2. Add a user (e.g. PerfectCollectionBot) with Admin role.
    3. Link the system user to your app asset and assign Full Control permissions.
    4. Click Generate New Token and select whatsapp_business_messaging.
    5. Save this token to WHATSAPP_TOKEN in your production environment settings.

    8.2 — Register Your Production Phone Number

    Replace the default Meta sandbox test number with your business line:

    1. Go to WhatsApp Manager → Phone Numbers → click Add Phone Number.
    2. Enter your phone details and complete the OTP authentication call.
    3. Verify your business settings on Meta Business Manager (requires document validation).
    4. Update your environment configuration file with the new WHATSAPP_PHONE_NUMBER_ID.

    8.3 — Verify Template Deploy Status

    Check the status of your message template (perfectcollection_invoice) in your Meta Business account to ensure it is marked as active before launching.

    8.4 — Enable Unlimited Customer Delivery

    The Sandbox developer app can only deliver messages to pre-approved numbers. Once your production account is verified, you can send invoices to any customer phone number automatically.

    8.5 — Understand Messaging Limits

    New business numbers have a starting limit of 1,000 outgoing conversations per day. Meta automatically increases this daily volume limit as you maintain good quality delivery ratings.

    8.6 — WhatsApp API Pricing

    Meta charges for conversations based on categories. In India, utility messages (like invoices) cost approximately ₹0.35 per message. The first 1,000 service conversations are free each month.
